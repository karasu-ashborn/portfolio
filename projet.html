DCK

Infrastructure Docker avec Docker Compose

Production

Objectif du projet

Déployer une infrastructure conteneurisée complète sur un serveur Debian pour héberger plusieurs services avec isolation réseau, reverse proxy et gestion des permissions. Architecture modulaire et reproductible avec Docker Compose.

Description

Mise en place d'une stack Docker Compose multi-services comprenant :

  • Configuration de plusieurs conteneurs avec docker-compose.yml
  • Reverse proxy Nginx pour le routage HTTP/HTTPS
  • Gestion des volumes pour la persistance des données
  • Réseau bridge personnalisé pour l'isolation
  • Gestion des permissions avec PUID/PGID pour éviter les conflits
  • Logs centralisés et monitoring des conteneurs

Technologies utilisées

Docker Docker Compose Nginx Debian Volumes Networks

Compétences développées

Conteneurisation Orchestration Reverse Proxy Gestion volumes
DNS

Pi-hole sur NAS avec réseau Macvlan

Production

Objectif du projet

Déployer Pi-hole comme serveur DNS de filtrage publicitaire et trackers sur un NAS, avec une adresse IP dédiée sur le réseau local via Macvlan pour permettre à tous les périphériques du réseau d'en bénéficier.

Description

Installation de Pi-hole en conteneur Docker avec configuration réseau avancée :

  • Déploiement Docker de Pi-hole sur NAS Synology/QNAP
  • Configuration réseau Macvlan pour IP dédiée (ex: 192.168.1.200)
  • Paramétrage DNS pour tout le réseau domestique
  • Blocage publicités, trackers et domaines malveillants
  • Dashboard de monitoring en temps réel
  • Listes de blocage personnalisées

Technologies utilisées

Pi-hole Docker Macvlan DNS NAS dnsmasq

Compétences développées

DNS Réseaux avancés Filtrage réseau Conteneurisation
MON

Supervision réseau avec PRTG & Zabbix

Production

Objectif du projet

Mettre en place une solution de supervision complète pour monitorer l'infrastructure réseau, les serveurs et les services. Création de dashboards personnalisés et alertes automatisées pour détecter rapidement les problèmes.

Description

Déploiement de deux solutions de monitoring complémentaires :

  • PRTG Network Monitor : Supervision serveurs Windows/Linux, switchs, services réseau
  • Zabbix : Monitoring avancé avec agents, logs, métriques système
  • Configuration capteurs SNMP pour équipements réseau (switch L3, routeur)
  • Surveillance disques, CPU, RAM, bande passante
  • Alertes email/SMS en cas de seuil dépassé
  • Dashboards temps réel pour visualisation globale
  • Historisation des données pour analyse

Technologies utilisées

Zabbix PRTG SNMP Agents MySQL PHP

Compétences développées

Supervision SNMP Alerting Dashboards Métriques
SRV

Serveur LAMP avec Proxy Squid

Production

Objectif du projet

Déployer une infrastructure web complète sur Debian avec stack LAMP (Linux, Apache, MySQL, PHP) pour héberger des applications web, couplée à un proxy Squid pour le contrôle et la journalisation du trafic HTTP/HTTPS.

Description

Installation et configuration d'un serveur web sécurisé avec proxy :

  • Stack LAMP : Apache2, MySQL/MariaDB, PHP 8.x, phpMyAdmin
  • Configuration VirtualHosts Apache pour multi-sites
  • Sécurisation Apache (mod_security, SSL/TLS)
  • Proxy Squid : Filtrage web, cache HTTP, contrôle d'accès
  • Journalisation centralisée avec rsyslog
  • Analyse logs avec logwatch
  • Sauvegarde automatisée des bases de données

Technologies utilisées

Apache MySQL PHP Squid rsyslog Debian

Compétences développées

Apache Bases de données Proxy Journalisation Sécurisation
